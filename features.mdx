## Automatic Resistance Control

SmartSpin2k's core feature is automatic resistance control through precise stepper motor positioning. The system responds to commands from your training app and adjusts your bike's resistance knob in real-time.

### How It Works

The stepper motor is controlled by a TMC2208/2209 driver with advanced features:

- **Precise positioning**: Tracks exact knob position in steps
- **StealthChop mode**: Quiet operation during normal riding
- **Stall detection**: Automatically detects end stops for homing
- **Temperature monitoring**: Adjusts current if ESP32 gets warm

From `Main.h:36`:
```cpp
int32_t targetPosition;   // The position the stepper will move to
int32_t currentPosition;  // Current stepper position

void moveStepper();       // Main stepper control function
```

### Resistance Modes

SmartSpin2k supports multiple resistance control modes:

1. **Simulation Mode** - Responds to incline/gradient changes from apps like Zwift
2. **ERG Mode** - Maintains target power output by adjusting resistance
3. **Resistance Mode** - Direct resistance level control (0-100%)
4. **Manual Shifters** - Override control with physical buttons

<Note>
  The incline multiplier setting (default: 5.0) controls how aggressively resistance changes with gradient. Adjust this in the web interface to match your fitness level.
</Note>

## ERG Mode (Electronic Resistance Control)

ERG mode is essential for structured training workouts. It maintains a constant power output by automatically adjusting resistance as your cadence changes.

### ERG Mode Operation

From `ERG_Mode.h:17`:
```cpp
struct Mode {
  static const int MAINTAIN   = 0;  // Holding target power
  static const int DECREASING = 1;  // Reducing resistance
  static const int INCREASING = 2;  // Increasing resistance
};
```

The ERG algorithm:
- Uses PID control loop for smooth adjustments (as of v25.4.8)
- Monitors cadence and adjusts resistance to hit target watts
- Includes feed-forward control for faster response
- Stops resistance if cadence drops below 30 RPM
- Resets to minimum resistance if you stop pedaling

### ERG Sensitivity

You can adjust ERG sensitivity in settings to control how aggressively the system responds:
- **Higher values**: More aggressive resistance changes
- **Lower values**: Smoother, more gradual adjustments

<Tip>
  ERG mode works best when combined with a calibrated power table. The system uses your power curve to make intelligent resistance decisions.
</Tip>

## Power Table Calibration

Most spin bikes have inaccurate or non-existent power meters. SmartSpin2k's power table system creates a calibration curve that maps resistance positions to actual power output.

### How Calibration Works

1. **Connect a reference power meter** (cycling power pedals or crank-based PM)
2. **Ride normally** while SmartSpin2k records data
3. **The power table learns** the relationship between resistance position, cadence, and watts
4. **Accurate power readings** are calculated from the calibration curve

From `Power_Table.h:24`:
```cpp
// Pick up new power value and put them into the power table
void processPowerValue(PowerBuffer& powerBuffer, int cadence, Measurement power);

// returns target position for given cadence and watts
int32_t lookup(int watts, int cad);

// returns watts for given cadence and target position  
int32_t lookupWatts(int cad, int32_t targetPosition);
```

### Power Table Features

- Automatic entry validation and duplicate removal
- Cubic spline interpolation for smooth curves
- Cadence compensation (normalizes to 90 RPM)
- Export/import via web interface or BLE
- Persistent storage in LittleFS filesystem

<Warning>
  The power table is reset when you change physical hardware (different resistance knob position, new resistance spring, etc.). You'll need to recalibrate.
</Warning>

## Bluetooth Low Energy Services

SmartSpin2k acts as a BLE server, broadcasting multiple standard fitness services simultaneously. This allows connection to training apps while also connecting to external sensors.

### BLE Services Provided

From the source code in `src/BLE_*.cpp`:

#### 1. Fitness Machine Service (FTMS)
- **Indoor Bike Data**: Power, cadence, speed, distance
- **Training Status**: Workout state, elapsed time
- **Supported Resistance Range**: Min/max resistance capabilities
- **Control Point**: Receives commands from training apps

#### 2. Cycling Power Service (CPS)
- **Power Measurement**: Instantaneous watts
- **Crank Revolution Data**: Cadence calculation
- **Sensor Location**: Crank-based power meter emulation

#### 3. Cycling Speed & Cadence (CSC)
- **Crank Revolutions**: Real-time cadence data
- **Wheel Revolutions**: Calculated speed (for apps that need it)

#### 4. Heart Rate Service
- **Pass-through**: Relays HR data from connected monitor
- **BLE Heart Rate Monitor support**: Connects to any standard HR strap

#### 5. Custom SmartSpin2k Characteristic

A proprietary characteristic for advanced configuration and control.

## Firmware Updates (OTA)

Over-The-Air (OTA) updates allow firmware upgrades without USB connection.

### Update Methods

1. **Automatic Updates** (if enabled)
   - Checks GitHub releases on boot
   - Downloads and installs latest firmware
   - Updates both firmware and web interface files

2. **Manual Upload**
   - Upload `.bin` firmware files via web interface
   - Upload individual `.html` or `.css` files
   - Useful for beta testing or custom builds

3. **USB Flash** (via PlatformIO)
   - For development and recovery
   - Requires USB cable connection

From `CHANGELOG.md` (v22.10.8):

<Warning>

</Warning>

## Automatic Homing

Homing calibrates the stepper position to known minimum and maximum resistance positions.

### Homing Methods

1. **End Stop Detection** (mechanical homing)
   - Motor turns until it detects resistance (stall)
   - Sets minimum and maximum step positions
   - Ensures motor doesn't exceed physical limits

2. **FTMS Resistance Feedback** (for smart bikes)
   - Uses reported resistance from connected bike
   - Maps resistance range to stepper positions
   - No mechanical contact needed

3. **Calibrate Trainer Command**
   - Triggered from training apps (Zwift spin-down)
   - Runs full homing routine
   - Updates min/max positions

From `Main.h:56`:
```cpp
void _findEndStop(bool moveForward);
void _findFTMSHome(bool bothDirections = false);
void goHome(bool bothDirections = false);
```

<Note>
  Homing is automatically performed on first boot. You can manually trigger it by requesting a calibration in your training app or pressing both shifters for 3 seconds.
</Note>

## Power Correction Factor

If you don't have a reference power meter for full calibration, you can use a simple power correction factor.

- **Range**: 0.5 to 2.5
- **Default**: 1.0 (no correction)
- **Use case**: Multiply reported power by this factor

Example: If your bike reads 200W but you know you're producing 220W, set factor to 1.10.

<Note>
  Power correction is a simple scalar multiplication. For best accuracy, use the full power table calibration instead.
</Note>

## Advanced Configuration

### External Control Mode

From `CustomCharacteristic.md:77`:
> BLE_externalControl: Disables internal calculation of targetPosition

Allows external systems to directly control stepper position via BLE, bypassing all internal logic.

### Sync Mode

From `CustomCharacteristic.md:82`:
> Forces stepperPosition = targetPosition to prevent motor movement during calibration

Useful for setting parameters without moving the motor.

### Simulation Modes

Individually enable/disable simulation for:
- Power (watts)
- Cadence (RPM)  
- Heart rate (BPM)
- Speed (kph)

When simulation is enabled, SmartSpin2k generates synthetic data instead of reading from sensors.

## Hardware Revisions

SmartSpin2k has evolved through three hardware versions:

### V1 - Archive (Original Design)
- Breadboard/prototype PCB
- External stepper driver
- Universal mounting system

### V2 - Through Hole (Current Recommended)
- Custom through-hole PCB
- Integrated TMC2208 driver socket
- Multiple mounting options
- Easy to solder for DIY builders

### V3 - Integrated PCB (Latest)
- SMT components
- Compact integrated design  
- Professional appearance
- Requires reflow soldering

<Tip>
  V2 through-hole design is recommended for DIY builders. V3 is best for commercial kit production or if you have SMT assembly capabilities.
</Tip>
